Things to Know

What is an API?

An API (Application Programming Interface) is a set of interfaces (in this case, url links) that are publicly available, that allow developers to interact with an application. Through this interaction with the interfaces, developers are able to send and receive data. For our usage, we will be using a URL interface in order to receive data.

API via URL links

There are some very popular and common APIs that developers use, such as Google Maps, YouTube and Twitter. Each of these APIs use URLs to get and post data.

For the Pokeapi, there are many URLs that can get data. By using a URL, such as https://pokeapi.co/api/v2/item/{id or name}, and combining it with a script call, we can get items from the Pokémon game by either entering an id or name. A more detailed example of an API call will be found below.

Ajax

The main reason why we are using jQuery is so that we can use a function called AJAX that simplifies making API calls. In JavaScript, you would need to write additional code to setup an API call, but with the jQuery.ajax() function, the setup is much simpler.

What is Ajax?

Ajax stands for Asynchronous JavaScript and XML. Ajax requests are triggered by JavaScript code; your code sends a request to a URL, and when it receives a response, a callback function can be triggered to handle the response. Because the request is asynchronous, the rest of your code continues to execute while the request is being processed, so it's imperative that a callback be used to handle the response.

Using jQuery.ajax()

In the code below, you can see how easy it is to make an API call using jQuery. The jQuery Ajax API accepts quite a number of optional arguments, but the minimum requirement is actually just a URL. The default method value is "GET" and the default dataType is "json". 


$.ajax({
    url: "test.html",
    method: "GET",
    dataType: "json"
})
    .done(function(){})
    .fail(function(){});

GET Request

The GET request is a Hyptertext Transfer Protocol (HTTP) method. HTTP is designed to enable communications between clients and servers. When making API calls, the client is the browser and the server will be the Pokéapi server. When using the GET method, we are requesting data from a specified source.

Structure

When making a GET request, we use a URL which identifies the specified source. For example, we will be using the following URL for one of the API calls later on in this guide.

https://pokeapi.co/api/v2/pokedex/1/?name=national
The structure consists of the API URL path plus a query string consisting of name and value pairs
Endpoint: https://pokeapi.co/api/v2/pokedex/1/
Name/value pair: ?name=national

JSON

JSON, or JavaScript Object Notation, is a minimal, readable format for structuring data. It is used primarily to transmit data between a server and web application, as an alternative to XML.

JSON Object

A JSON object is made up of a key and value pair. A key is always enclosed in quotation marks and a value can be a string, number, boolean, array or object.
